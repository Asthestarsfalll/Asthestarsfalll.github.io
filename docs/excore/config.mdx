:::info
ExCore 重新思考了当下配置文件及各类开发套件的繁冗之处，选择了 `toml` 作为配置文件，并且提供了简洁的接口构建对象，同时引入了 ConfigHook 增强了整个配置系统的扩展性。
:::

## Overview

配置系统的最主要功能就是通过 `配置文件` 来 `创建对象`， 而在所有创建的对象之中，ExCore 将其分为三类——与模型训推之间相关的`主要对象` 、间接相关的`孤立对象` 和创建过程的`中间对象`

- 主要对象包括—— 模型、损失函数、Dataset、优化器、学习率调度器等。
- 孤立对象则包括一些训练迭代数、日志地址，这些对象大多在读取配置文件的阶段就被加载为python 对象。
- 中间对象指的是某些对象创建的过程中所需要创建的其他对象，如一个分割网络，需要主干网络和分割头，则该分割网络是`主要对象`，主干和分割头则是`中间对象`。

:::note
我们往往并不需要获得这些中间对象，但某些时候还是需要它们，万幸， ExCore 提供了 ConfigHook 来进一步操控这些对象的创建流程。
:::

因此`excore.config` 仅提供两个接口———[load](#load) 与 [build_all](#build_all)，load 将加载配置文件为字典，build_all 则直接一次性创建所有对象，并返回包含上述的两个`主要对象`和`孤立对象` 的字典。

在具体介绍 API 前，首先应该了解 ExCore 的配置系统是如何工作的。

| 名称   |                       说明                       |

| 组件   |     即一个对象，如ResNet、Adam、COCODataset      |
| 类型   |     不同类型的组件会被注册进不同的注册器当中     |



### 配置文件

相较与使用 `yaml` 和 `py` 作为配置文件，被称为最强配置文件的 [toml](https://toml.io/cn/v1.0.0) 的层次更为一目了然，且编写更加方便。

样例：

```toml
[Model.ResNet]
in_channels=3
depth=50

[Optimizer.Adam]
lr=1e-3
```

:::note
这里我们以 `[]` 区分不同类型的组件，在方括号中`.`后的名称 `ResNet` 和 `Adam` 即为 注册器 中的被注册的类/函数。创建时，ExCore 会自动地搜寻这些名称进行创建。
:::

:::tip
上述写法虽然有很多等价表达，但是仍然推荐使用方括号的编写方式。
:::

ExCore 中对 toml 支持的格式进行了一些细微的修改，以适应 `某个组件的`
