:::info
类似于当前绝大多数开发套件，ExCore使用 注册器 (Registry) 来管理各种模块，如模型的 主干 、头部、 数据集模块等.当然， ExCore 也支持将全部模块都注册在一起.

注册器 其实就是字符串到类或函数的映射，可以配合 配置文件便捷地创建对象或调用函数.
:::
```python
class Registry(dict)：
	children: Dict[str, "Registry"] = dict()
```
Registry 类继承自 `dict`，并且拥有类属性 `children` 用来存储已创建的 Registry 对象（下文称之为 注册器）.

## \_\_new\_\_
```python
def __new__(cls, name: str, extra_field: Any = None) -> "Registry"
```
:::danger
Registry 重写了 \_\_new\_\_ 方法，对于相同 name 的注册器只会创建一次，并储存在类属性 `children` 中，再次调用只会返回已经创建好的实例，即使它们的 `extra_field` （见[\_\_init\_\_](#\_\_init\_\_)）参数并不相同.
:::
:::danger
以双下划线 \_\_ 开头的 name 并不遵从上述规则，而是每次都会重新创建.
:::
## \_\_init\_\_

```python
def __init__(
	self, 
	name: str, 
	*, 
	extra_field: Optional[Union[str, Sequence[str]]] = None
) -> None
```

当我们创建一个 注册器时，除了要指定其名称（name）外，还可以传入一些额外信息，如是否拥有预训练，是否是主干网络等.`extra_field` 便储存了这些额外信息的名称.

样例：

```python
from excore import Registry
MODELS = Registry("model"， extra_field=['is_pretrained', 'is_backbone'])
```

:::note
传入 extra_field 参数时，必须要使用关键字参数.
:::

我们可以通过 filter 方法来寻找符合一定规则的组件的名称，如获得全部拥有预训练模型的组件，具体见 [filter](#filter)：
```python
from excore import Registry
MODELS = Registry("model"， extra_field=['is_pretrained', 'is_backbone'])
# 一系列注册之后
# ...
# ...
filtered_model_name = MODELS.filter('is_pretrained')
```
## register
```python
def register(
	self, 
	force: bool = False, 
	name: Optional[str] = None, 
	**extra_info,
) -> Callable
```

:::note
  创建注册器时传入的 name 参数只能包含数字、字母和下划线.
:::

:::note
当有两个组件的 `name`一致时，默认不会进行覆盖，而是抛出异常.如需进行覆盖请指定 force 参数
:::

样例：
```python
from excore import Registry
# 创建一个名为 model 注册器
# 后面我们可以通过 model 找到该注册器
MODELS = Registry("model")

# 使用装饰器进行注册
@MODELS.register()
class ResNet:
	pass

# 当没有指定 name 时，注册器的键名即为 函数/类 的名称
# 本例中即 get_resnet50
@MODELS.register(name='ResNet50')
def get_resnet50():
	return ResNet(...)
```


上述代码注册了两个组件：
```
'ResNet' -> <class 'ResNet'>
'ResNet50' -> <function 'get_resnet50'>
```


如果我们在其他文件中还有一个`ResNet`需要注册：
```python
from excore import Registry
# 会返回已经创建过的注册器
MODELS = Registry("model")

# 需要指定 force 参数进行覆盖
@MODELS.register(force=True)
class ResNet:
	pass
```

注册时添加额外信息：
```python
from excore import Registry
#  创建一个注册器
MODELS = Registry("model", extra_feild='xxx')

# 注册
@MODELS.register(xxx=True)
def VGG:
	pass
```

:::note
extra_info 的 键 必须存在于 extra_field 中，若未传入extra_info，则默认为 None
:::

:::tip
如果某些组件来自外部包，不能通过装饰器的方式进行注册，您可以直接调用`_register`方法，如：
```python
from excore import Registry
from xxx import ResNet
# 会返回已经创建过的注册器
MODELS = Registry("model")
MODELS._register(ResNet, force=False, name=None)
```
或者调用[register_all](#register_all)方法.
:::

## register_all
```python
def register_all(
	self, 
	modules: Sequence[Callable], names: Optional[Sequence[str]] = None, 
	extra_info: Optional[Sequence[Dict[str, Any]]] = None, 
	force: bool = False,
) -> None:
```
该方法用于批量注册组件（如果您不想使用装饰器的话）.

样例：

```python
from excore import Registry
from xxx import ResNet, VGG, SENet
MODELS = Registry('model', extra_field='is_backbone')
MODELS.register_all(
	[ResNet, VGG, SENet],
	# 请务必保证names和extra_info参数长度和传入组件长度相同
	names = ['a', 'b', 'c'],
	extra_info = [
			{'is_backbone': True},
			{'is_backbone': True},
			{'is_backbone': True},
		],
	force = False,
)
```

## merge
```python
def merge(
	self, 
	others: Union["Registry", List["Registry"]], 
	force: bool = False,
) -> None
```

用于合并两个或者多个 注册器

样例：

```python
MODELS.merge([BACKBONES, HEADS])
```

:::note
merge 暂时并不支持合并 `extra_info`.
:::

## fuzzy_match

```python
def fuzzy_match(self, base_module, match_func=_default_match_func)
```

按照传入的匹配方法注册指定模块内的相应的函数/类.

匹配方法应该拥有两个参数，分别代表函数/类的名称和上文指定的模块.

默认方法为：

```python
def _default_match_func(m, base_module):
    if not m.startswith("__"):
        m = getattr(base_module, m)
        if inspect.isfunction(m) or inspect.isclass(m):
            return True
    return False

```

样例：

```python
from xxx import transforms
TRANSFORMS = Registry("Transform")

TRANSFORMS.fuzzy_match(transforms)
```


## filter

```python
def filter(
	self, 
	filter_field: Union[Sequence[str], str], 
	filter_func: Callable = _default_filter_func,
) -> List[str]
```

按照某种方式过滤 注册器 中的 extra_info，传入需要过滤的 field，以及判断是否留下的 filter_func (可选).
:::note
filter_func应该是 一个 拥有一个序列输入的函数，当未传入 filter_func时，调用默认的过滤方法，如下
```python
def _default_filter_func(values: Sequence[Any]) -> bool:
    for v in values:
        if not v:
            return False
    return True
```
即检查传入序列的所有元素是否是 python 中有效的（None, {}, []等均为False）.
:::

样例：

```python
from excore import Registry
MODELS = Registry('Model' extra_field = ['is_pretrained', 'is_backbone'])
#  一系列注册
# ...
# ...
filtered_module_name = MODELS.filter('is_pretrained')
```

当我们想找到 `is_pretrained=True` 而 `is_back_bone=False` 有哪些模型时：

```python
from excore import Registry
MODELS = Registry('Model' extra_field = ['is_pretrained', 'is_backbone'])
#  一系列注册
# ...
# ...
def _check_func(values):
	if values[0] and not values[1]:
		return Ture
	return False

filtered_module_name = MODELS.filter(['is_pretrained', 'is_backbone'], _check_func)
```

:::danger
为了避免各种意料之外的事，传入顺序请务必按照初始化时传入的field顺序.
:::

## module_table

```python
def module_table(
	self, 
	filter: Optional[Union[Sequence, str]] = None, 
	select_info: Optional[Union[Sequence[str], str]] = None, 
	module_list: Optional[Sequence[str]] = None, 
	**table_kwargs
) -> Any
```

用于返回 注册器 已经注册的组件及其额外信息（extra_info），支持搜索功能.

:::note
支持自定义表格样式，需要熟悉 tabulate 的使用方法
:::

样例：
假设这里有一个 注册器

```python
from excore import Registry
# 返回已经创建好的 注册器， 见 `get_child` 小节
MODELS = Registry.get_child('Model')
print(MODELS.module_table())
```
结果如下：
```
╒════════════╕  
│ Model      │  
╞════════════╡  
│ ResNet18   │  
├────────────┤  
│ ResNet50   │  
├────────────┤  
│ ResNet101  │  
├────────────┤  
│ VGG        │  
├────────────┤  
│ SENet      │  
╘════════════╛
```

:::note
第一行为 注册器 的 name，上例中为 `Model`
:::

当我们只想搜索某一个网络时，可以指定`filter`参数，`module_table`会使用`fnmatch`进行匹配.

```python
from excore import Registry
# 返回已经创建好的 注册器， 见 `get_child` 小节
MODELS = Registry.get_child('Model')
	print(MODELS.module_table(filter=['**ResNet**']))
```
结果如下：
```
╒════════════╕  
│ Model      │  
╞════════════╡  
│ ResNet18   │  
├────────────┤  
│ ResNet50   │  
├────────────┤  
│ ResNet101  │  
╘════════════╛
```

:::tip
当需要搜索多个组件时，可以传入`filter`列表.如\['\*\*ResNet\*\*', '\*\*VGG\*\*\]
:::

当我们想展示额外信息时：

```python
from excore import Registry
# 返回已经创建好的 注册器， 见 `get_child` 小节
MODELS = Registry.get_child('Model')
	print(MODELS.module_table(select_info=['is_backbone']))
```

结果如下：
```
╒════════════╤═══════════════╕  
│ Model      │ is_backbone   │  
╞════════════╪═══════════════╡  
│ FCN        │               │  
├────────────┼───────────────┤  
│ ResNet     │ True          │  
├────────────┼───────────────┤  
│ SimpleHead │ False         │  
╘════════════╧═══════════════╛
```

:::note
上述结果中空行即结果为None，可能是注册时未传入该 额外信息，或是传入信息为None.
:::


## get_child

```python
@classmethod
def get_child(cls, name: str, default: Any = None) -> Any
```

Registry 会将合法（非双下滑线开头）名称（`name`）的组件添加至类属性 `children` 中，除了使用上面提到的方法，直接使用get_child也是等价的.

```python
from excore import Registry
MODELS = Registry('model')
MODELS = Registry.get_child('model')
```

:::tip
为了显式的区分是否已创建当前 注册器， 降低代码阅读难度，推荐使用 `get_child` 方法.
:::

## find
```python
@classmethod
def find(cls, name: str) -> Any
```

在 `children` 的所有 注册器 中寻找名为 注册名 为 name 的组件.返回该组件及其所属注册器的名称.

样例：

假设现在有两个注册器 MODELS 和 HEAD , 其中 ResNet 在 MODELS 中：
```python
from excore import Registry
MODELS = Registry('model')
HEADS = Registry('head')
ResNet, base_name = Registry.find('ResNet')
```
其中 ResNet 为 <class ResNet\>， base_name为 注册器 MODELS 的属性 `name` —— model.

## make_global

```python
@classmethod
def make_global(cls) -> "Registry"
```

将 `children` 中所有的注册器合并，返回一个拥有全局映射关系的 Registry 对象.
:::note
该对象名称为 \_\_global，当前版本合并时extra_info并不会合并.
:::
